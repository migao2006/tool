# Work Package Template — Not Executable
## Status
TEMPLATE
## Authorization
Default to `FULL_AUTONOMY_UNTIL_MAIN_UPDATE`; record any explicit narrower grant.
## Primary Outcome
State one natural, complete, reviewable outcome.
## Background
Record only verified current facts and why the outcome is needed.
## Subtasks
List the tightly related work required to finish the outcome.
## Allowed Scope
Name responsible modules and evidence-based direct callers or support files.
## Prohibited Changes
Record task-specific exclusions and the final protected-branch update boundary.
## Public Contracts
Reference authoritative contracts and list any explicitly authorized change.
## Risk Classification
Set LOW, MEDIUM, HIGH, or CRITICAL and give the evidence.
## Validation Plan
List exact focused, lint/type, diff, Fast, Full, and review checks that apply.
## Stop Conditions
List concrete conditions that require PARTIAL or BLOCKED reporting.
## Definition of Done
List objectively verifiable completion criteria for the whole Work Package.
## Results
Record actual results only during execution; never leave template guidance here.
