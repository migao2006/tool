## 月營收完整度

上市月營收的官方當期橫斷面回傳 945 檔；以該來源對照的 1,083 檔上市公司清單計算，最初缺少 138 檔。這個 1,083 檔的「公司清單」口徑與上表經證券分類後的 1,082 檔「上市選股母體」口徑不同，不應直接混用。

系統已改為在可用配額內，對官方橫斷面缺漏的股票逐檔查詢歷史月營收。截至本次最終稽核，上市快照已由最初缺少 138 檔降為 130 檔；其中 63 檔通過流動性／風險預篩，會進入深度修復佇列，另 67 檔位於選股預篩之外，保留「官方橫斷面未列」診斷但不占用有限的深度 API 配額。上櫃最新快照的月營收缺漏為 0 檔。

缺漏必須依下列原因分類，不能一律當成 0 分或 0 元：

| 類型 | 判定方式 | 系統處理 |
| --- | --- | --- |
| 官方橫斷面省略、可逐檔補回 | 當期上市月營收總表沒有該公司，但個股歷史端點可取得資料 | 排入優先補齊佇列；補回前降低資料信心，不把營收視為 0 |
| ETF 不適用 | 標的分類為 ETF | 不要求公司月營收、EPS、ROE 或公司現金流，也不列為資料缺失 |
| 上游／API 錯誤 | 官方來源與逐檔來源都失敗、逾時，或回傳格式無法驗證 | 保留最近一次可驗證資料、記錄來源診斷與重試狀態；不以臆測值補齊 |

因此，目前上市仍缺少的 130 檔不等同於公司沒有營收：63 檔是可逐檔修復的候選資料，67 檔是預篩外標的的已知官方橫斷面缺口。ETF 沒有公司月營收則是設計上不適用，不是 API 故障。

## 深度資料驗證

- 股價歷史已更新至 2026-07-14。
- 實測的上櫃與 ETF 樣本均可取得至少 280 筆日行情，技術面不再因代號格式或錯誤的還原行情端點而整批失效。
- 正式排行榜已累積 99 檔 v16.3 最後成功驗證結果（上市 18、上櫃 17、ETF 64）；另有上市 12、上櫃 13 檔 v16.2 舊列，會依新版候選佇列自動重算，但不混入 v16.3 排名。交易日切換不再把同版本的前一日 last-good 結果整批藏起來，每張卡會標示驗證日期。
- 公司財務資料會分別保存月營收、季度營收與現金流；ETF 不套用公司財報評分。
- 資料來源期間落後、格式不適用或欄位缺少時，會寫入來源診斷並降低信心度，不把缺值默認為 0。
- 目前 27 檔 v16.3 公司結果已排入一次性來源覆蓋重驗；這是為了補上舊快取缺少的 income／balance／cashflow 覆蓋標記，不代表 27 檔都已確認 API 故障。另有 25 檔 v16.2 舊列會因模型版本不相容直接視為未驗證候選，不重複計入修復數。重抓後會再依 `available`、`source-not-returned` 或 `source-not-comparable` 明確分類。ETF 的公司財報欄位不適用，因此不會進入公司財報修復佇列。
- ETF 方向辨識不再直接使用 MOPS 的「槓桿／反向」基金大類；該欄是共用類別，不能代表實際方向。系統改以基金名稱、備註與追蹤指數判斷，並已將 11 檔曾同時被標成正向槓桿與反向的舊快取排入立即重算。

上述結果只代表已完成的第一批正式環境驗證；後續批次仍由同步狀態、來源診斷與資料信心持續監控。

## API 配額與同步效率

目前正式環境未設定 FinMind token，因此採用實際可用的每 60 分鐘 300 單位上限。所有工作共用資料庫中的原子配額帳本：每批在發送請求前先以交易方式保留單位，避免上市、上櫃與 ETF 排程同時執行而超額。

目前排程（台灣時間，Asia/Taipei）為：

| 工作 | 排程 |
| --- | --- |
| 市場母體更新 | 週一至週五 14:43 |
| 上市深度同步 | 每小時 01、21、41 分 |
| 上櫃深度同步 | 每小時 08、28、48 分 |
| ETF 深度同步 | 每小時 15、35、55 分 |

批次會依當下滾動 60 分鐘剩餘額度自動縮小，而不是超過限制後才等待 API 拒絕。公司股同步亦會交錯安排「月營收缺漏修復」與一般高優先序候選，避免缺漏修復完全擠掉正常累積，或正常候選讓缺漏長期無法補齊。

HTTP 200 但內容為空、來源期別落後的公司資料會另設 `needs_repair`，每 6 小時重新進入最多四分之一批次的修復槽；最後成功資料仍可顯示，不會因一次空回應整檔消失。財報快取只有在損益表、資產負債表與現金流量表三者都有覆蓋時才允許沿用。

點時回測快照另設覆蓋門檻：上市、上櫃、ETF 各自至少完成 75% 候選驗證才會封存。未達門檻只更新網站備援資料，不會成為回測樣本，避免先驗證高分股造成樣本選擇偏誤。

## 驗收結論

本次可確認：市場母體、TDCC、日行情、上市／上櫃分類、ETF 分流、月營收診斷與中央配額控制均已在正式環境跑通。仍待補回的上市月營收已被明確標記並排程修復；系統不會把這些缺漏當成零營收，也不會把 ETF 的不適用欄位誤報成 API 缺失。
